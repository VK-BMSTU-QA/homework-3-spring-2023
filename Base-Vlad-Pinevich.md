# [Cтраница товара](https://www.reazon.ru/product/57)
### Название товара
![item-title](https://user-images.githubusercontent.com/57019979/221987819-798f374a-6e14-4f9d-a2a5-9f158373fcf1.png)
 - Название помещается целиком
 - При клике на название загружается страница этого же товара
 - Категория соответствует категории данного товара
    - Категория товара - как правило первое слово или два в названии товара
    - В ином случае название товара непосредственно ассоциируется с категорией, указанной в блоке, на скрине ниже
 ![image](https://user-images.githubusercontent.com/57019979/224577916-f2be5d95-a041-4006-9456-9b5dcc9946de.png)

 - При клике на категорию загружается [страница категории](https://www.reazon.ru/category/accessories)
 - Рейтинг товара
     - Если нет отзывов, то звездочка серая и отображается рейтинг 0.0
     - Если есть отзывы, то звездочка желтая
     - По клику открывается страница отзывов
 - Отзывы
     - Если нет отзывов, отображается надпись "Нет отзывов"
     - Если есть отзывы, то отображается их количество в формате "Х отзывов"
     - По клику открывается страница отзывов
 - Избранное
     - Если пользователь неавторизирован появляется сообщение об ошибке при клике
     - Если пользователь авторизирован, то товар добавляется в избранное и иконка красится в розовое при клике
     - Если товар уже добавлен в избранное, то иконка розовая, 
     - Если товар еще не добавлен в избранное, то незакрашенная с черной гранью

### Создание отзыва
![image](https://user-images.githubusercontent.com/57019979/222456421-d88e3910-3d91-4f14-8b2f-f6565cfdc1d8.png)
 - Выбор рейтинга
    - Звездочки загораются при наведении курсора, если рейтинг не выбран. 
    - Если рейтинг выбран, то звездочки загораются при наведении курсора только для более высокого рейтинга, чем выбран ранее. 
    - Без выбора рейтинга нельзя отправить отзыв, будет появляться соответствующее сообщение об ошибке
 -  Ограничений на символы нет, но есть ограничение по длине текста в каждом поле в 300 символов. 
 -  Разрешается оставлять пустые поля ввода
 -  При попытке отправить строки с целью эксплуатации XSS угрозы, отзыв оформлется, но потенциально вредностные символы в отзыв не попадают и скрипт не исполняется
 -  При клике на кнопку "Отправить отзыв" создается отзыв и открывается страница просмотра [отзывов](https://www.reazon.ru/comment/241) данного товара

# Хедер
![image](https://user-images.githubusercontent.com/57019979/221990770-23311a5a-be0d-4764-b995-568764dbeee6.png)
- Логотип кликабельный, ведет на главную страницу
- Кнопка категории
   - При нажатии появляется гамбургер-меню с категориями товаров
   - При наведени на катеогрии они выделятся более темным цветом
   - *Баг.* Меню не скрывается при нажатии на кнопку каталога
   <img src="https://user-images.githubusercontent.com/57019979/222451895-61ee1068-e136-4e0a-a01e-c72b87d86fb7.png" style="height: 250px;"     alt="hamburger-selector">
  
     - *Корректное поведение.* Меню скрывается при нажатии в любое место, кроме самих категорий
- Поисковая строка
  - Для отправки поискового запроса требуется ввести 3 и более символов
  - Допускаются латинские и кириллические буквы, цифры и символы !?()_./-
  - Если поисковой запрос содержит другие символы или их менее 3, то появляется соответсвующее сообщение
     - Исключение. Если в поисковом запросе категория товара, то возможен поиск по любому числу симовлов
  - *Баг*. При вводе меньше 3 символов название вкладки отображается "- Reazon"
     ![image](https://user-images.githubusercontent.com/57019979/222171885-1dab6eea-ae7c-4664-8c18-0947630d6d56.png)
     - *Корректное поведение.* Название вкладки "Поиск - Reazon"
  - При вводе символов в инпут поиска появляется от 1 до 3 поисковой подсказки
    - При нажатии на саджест выполняется соответсвующий поисковой запрос
    - Подсказка отображается, только в случае, если введенный в инпут текст похож на название товара или категории

  - При нажатии открывается страница [избранного](https://www.reazon.ru/user/favorites)
- Корзина 
  - При нажатии открывается страница [корзины](https://www.reazon.ru/cart)

# [Cтраница профиля](https://www.reazon.ru/user)
#### Блок личной информации
![Блок личной информации](https://user-images.githubusercontent.com/59176894/222009807-db10a007-339e-443a-8901-a347c7d91a90.png)  
- Блок аватарки
  - При наведении на фотографию пользователя появляется кнопка добавления аватарки
  - При наведении на фотографию пользователя при добавленной аватарке появляется блок кнопок изменения и удаления аватарки
  - *Баг:* при наведении на фото курсор меняется на pointer, при клике ничего не происходит.  
    ![image](https://user-images.githubusercontent.com/59176894/224557266-40626bfc-f24c-4a0c-9420-25df908e2969.png)
    - *Корректное поведение:* курсор не меняется при наведении на фото
  - *Баг:* при клике на кнопку изменения аватарки и отмене выбора файла удаляется старая аватарка, а на ее месте остается аватарка по умолчанию, при этом кнопок все еще две - для изменения и удаления аватарки. Лечится перезагрузкой страницы
    ![image](https://user-images.githubusercontent.com/59176894/224557525-b108e7f7-2721-4c7a-a121-5a6956b1c019.png)
    - *Корректное поведение:* при отмене выбора файла аватарка остается той же, что была до нажатия на кнопку изменения

- Блок имени
  - При нажатии на иконку карандаша открывается поп-ап с полем для ввода нового имени и кнопками применения изменений и отмены  
    ![image](https://user-images.githubusercontent.com/59176894/222013367-ecc088c8-14d8-40ce-886c-97a0e4c4a0ac.png)
  - При клике по инпуту вокруг него появляется граница темного цвета, при клике вне инпута граница пропадает (далее для всех форм аналогично)
  - При нажатии "Применить" в случае пустого инпута справа от него появляется подсказка, что для применения необходимо написать новое имя  
    ![image](https://user-images.githubusercontent.com/59176894/222013284-802b1cf9-8dd3-4c27-afef-75f88a0f2927.png)
  - При нажатии "Применить" в случае инпута, заполненного пробелами, также появляется подсказка о необходимости заполнения поля
  - При нажатии "Применить" в случае непустого инпута новое имя применяется и меняется в блоке личной информации, при этом убираются пробелы в начале и конце введенной строки, при наличии нескольких пробелов между словами - остается один. Поп-ап закрывается
  - При нажатии "Отменить" поп-ап закрывается без изменений имени в случаях пустого и непустого инпута (далее для всех форм аналогично)

- Блок почты
  - При нажатии на иконку карандаша открывается поп-ап с полем для ввода новой почты и кнопками применения изменений и отмены
  - При нажатии "Применить" в случае пустого инпута справа от него появляется подсказка, что для применения необходимо заполнить поле
  - При нажатии "Применить" в случае инпута, заполненного пробелами, также появляется подсказка о необходимости заполнения поля
  - При нажатии "Применить" в случае инпута, заполненного строкой, не соответствующей базовому виду почты (text@text), появляется подсказка, что введенная почта имеет неверный формат  
    ![image](https://user-images.githubusercontent.com/59176894/222016516-60c22251-4f3f-4323-8838-02bbf3a4b508.png)
  - При нажатии "Применить" в случае введения валидной (regex: *@*) почты в инпут новая почта применяется и меняется в блоке личной информации, при этом убираются пробелы в начале и конце введенной строки. Поп-ап закрывается

- Блок телефона
  - Так как при регистрации нет поля для ввода номера телефона, до его заполнения пользователем там будет написано "Нет информации"  
    ![image](https://user-images.githubusercontent.com/59176894/222017172-bb5ee32e-9aa0-4e40-9cc4-989c8c384aa5.png)
  - При нажатии на иконку карандаша открывается поп-ап с полем для ввода номера телефона и кнопками применения изменений и отмены.  
    ![image](https://user-images.githubusercontent.com/59176894/222017290-6dd37b11-3e74-412c-a587-b1103a0f032a.png)
  - При нажатии "Применить" в случае пустого инпута справа от него появляется подсказка, что для применения необходимо заполнить поле
  - При нажатии "Применить" в случае несоответствующего требованиям (!= 11, включая автозаполненный код +7) количества цифр  справа от инпута появляется подсказка, что введено недостаточное количество цифр номера  
    ![image](https://user-images.githubusercontent.com/59176894/222017581-b6700cd0-53af-48a9-bbc5-c584f784634e.png)  
    ![image](https://user-images.githubusercontent.com/59176894/222018044-4ea40ac3-a690-439e-93a8-351a96b32ac3.png)
  - При нажатии "Применить" в случае, когда код региона введенного номера не равен +7, справа от инпута появляется подсказка, что телефон должен начинаться с +7  
  ![image](https://user-images.githubusercontent.com/59176894/222018280-44845e4c-844d-4287-8016-6110ce3b2c7d.png)
  - При нажатии "Применить" в случае соответствующего требованиям (11 цифр, начинающихся с "+7") заполнения инпута новый номер телефона применяется и меняется в блоке личной информации, при этом убираются пробелы в начале и конце введенной строки. Поп-ап закрывается

- Блок изменения пароля
  - При нажатии на ссылку "Изменить пароль" открывается поп-ап с полями для ввода старого и нового (дважды) пароля и кнопками применения изменений и отмены.  
    ![image](https://user-images.githubusercontent.com/59176894/222018848-3ddbf8eb-1290-4446-8f23-ee0dbb113e68.png)
  - При нажатии "Применить" в случае пустого инпута нового пароля справа от него появляется подсказка, что для применения необходимо заполнить поле
  - При нажатии "Применить" в случае несоответствующего требованиям (<6 символов) заполнения инпута нового пароля справа от него появляется подсказка с требованиями к паролю  
    ![image](https://user-images.githubusercontent.com/59176894/222019375-4aa24646-4f08-4c5a-bed3-666468dc9355.png)
  - При нажатии "Применить" в случае соответствующего требованиям (>=6 символов) заполнения инпута нового пароля и пустого инпута для повтора пароля справа от него появляется, что пароли должны совпадать  
    ![image](https://user-images.githubusercontent.com/59176894/222019586-afb1f030-5160-4935-8034-91fd945b4c96.png)
  - При нажатии "Применить" в случае соответствующего требованиям (>=6 символов, содержимое инпутов "Новый пароль" и "Повторите пароль" совпадает) заполнения инпута нового пароля и инпута для повтора пароля, но пустым полем для ввода текущего пароля справа него появится подсказка, что для применения необходимо заполнить поле
  - При нажатии "Применить" в случае соответствующего требованиям (>=6 символов, содержимое инпутов "Новый пароль" и "Повторите пароль" совпадает) заполнения инпута нового пароля и инпута для повтора пароля, но с неверно введенным текущим паролем справа сверху страницы появится сообщение о том, что введенный старый пароль неверный  
    ![image](https://user-images.githubusercontent.com/59176894/222020455-2588689e-bb7c-4b41-b118-8db04e56ac20.png)
  - При нажатии "Применить" в случае соответствующего требованиям (все пароли >=6 символов, новый пароль и повторенный новый пароль совпадают, текущий пароль введен верно) заполнения формы изменения применяются. Поп-ап закрывается
