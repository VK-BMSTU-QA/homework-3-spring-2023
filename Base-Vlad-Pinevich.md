# Общие паттерны

- Как проверить авторизирован ли пользователь
    Ищем элемент с классом `header__right`, в innerHTML ищем элемент `<a  href="/login"></a>`

- Как проверить появление сообщение об ошибке
    HTML-элемент c `id="header_error-messag"` меняет CSS свойство с `display: none;` на `display: flex;`, через три секунды оно меняется обратно. При этом innerText этого элемнта должен быть текст ошибки

### Создание отзыва
![image](https://user-images.githubusercontent.com/57019979/222456421-d88e3910-3d91-4f14-8b2f-f6565cfdc1d8.png)
 - Выбор рейтинга
    - Звездочки загораются при наведении курсора, если рейтинг не выбран. 
    - Если рейтинг выбран, то звездочки загораются при наведении курсора только для более высокого рейтинга, чем выбран ранее. 
    - Без выбора рейтинга нельзя отправить отзыв, будет появляться соответствующее сообщение об ошибке
 -  Ограничений на символы нет, но есть ограничение по длине текста в каждом поле в 300 символов. 
 -  Разрешается оставлять пустые поля ввода
 -  При попытке отправить строки с целью эксплуатации XSS угрозы, отзыв оформлется, но потенциально вредностные символы в отзыв не попадают и скрипт не исполняется
 -  При клике на кнопку "Отправить отзыв" создается отзыв и открывается страница просмотра [отзывов](https://www.reazon.ru/comment/241) данного товара

# Хедер
![image](https://user-images.githubusercontent.com/57019979/221990770-23311a5a-be0d-4764-b995-568764dbeee6.png)
- Логотип кликабельный, ведет на главную страницу
- Кнопка категории
   - При нажатии появляется гамбургер-меню с категориями товаров
   - При наведени на катеогрии они выделятся более темным цветом
   - *Баг.* Меню не скрывается при нажатии на кнопку каталога
   <img src="https://user-images.githubusercontent.com/57019979/222451895-61ee1068-e136-4e0a-a01e-c72b87d86fb7.png" style="height: 250px;"     alt="hamburger-selector">
  
     - *Корректное поведение.* Меню скрывается при нажатии в любое место, кроме самих категорий
- Поисковая строка
  - Для отправки поискового запроса требуется ввести 3 и более символов
  - Допускаются латинские и кириллические буквы, цифры и символы !?()_./-
  - Если поисковой запрос содержит другие символы или их менее 3, то появляется соответсвующее сообщение
     - Исключение. Если в поисковом запросе категория товара, то возможен поиск по любому числу симовлов
  - *Баг*. При вводе меньше 3 символов название вкладки отображается "- Reazon"
     ![image](https://user-images.githubusercontent.com/57019979/222171885-1dab6eea-ae7c-4664-8c18-0947630d6d56.png)
     - *Корректное поведение.* Название вкладки "Поиск - Reazon"
  - При вводе символов в инпут поиска появляется от 1 до 3 поисковой подсказки
    - При нажатии на саджест выполняется соответсвующий поисковой запрос
    - Подсказка отображается, только в случае, если введенный в инпут текст похож на название товара или категории
- Кнопка заказы
  - При нажатии открывается страница [заказов](https://www.reazon.ru/orders)
- Кнопка избранного
  - При нажатии открывается страница [избранного](https://www.reazon.ru/user/favorites)
- Корзина 
  - При нажатии открывается страница [корзины](https://www.reazon.ru/cart)
- Если пользователь не авторизован
  - Отображается кнопка войти
     ![image](https://user-images.githubusercontent.com/57019979/224579277-29ffd225-6c10-4958-9b8a-c3e084a22939.png)

       - При нажатии открывается страница для входа в аккаунт
- Если пользователь авторизован
  - Отображается кнопка Профиль
     ![image](https://user-images.githubusercontent.com/57019979/224579306-0277a357-391d-4a05-bc64-d60028aec5ea.png)
     
       - При нажатии появляется модальное окно с кнопкой выйти
         ![image](https://user-images.githubusercontent.com/57019979/224580471-2f39e414-69ec-4745-87c3-7cc46f125301.png)

           - Если нажать кнопку в модальном окне, то оно будет скрыто и пользователь разлогинен
           - Если нажать в любое другое место, то модальное окно будет скрыто
 - Хедер перерендривается только при авторизации или окончании авторизованной сессии
   - Инструкция для проверки какие компоненты [ререндерятся](https://github.com/tUnknownLegend/TP-QA-HW-1/edit/Base/Base.md#%D0%BA%D0%B0%D0%BA-%D0%BF%D1%80%D0%BE%D0%B2%D0%B5%D1%80%D0%B8%D1%82%D1%8C-%D0%BF%D0%B5%D1%80%D0%B5%D0%BD%D1%80%D0%B4%D1%80%D0%B8%D0%B2%D0%B0%D0%B5%D1%82%D1%81%D1%8F-%D0%BB%D0%B8-%D0%BA%D0%BE%D0%BC%D0%BF%D0%BE%D0%BD%D0%B5%D0%BD%D1%82)
   - Окончание сессии происходит при нажатии кнопки выйти 
   - Авторизация проиходит на страницах [логина](https://www.reazon.ru/login) и [регистрации](https://www.reazon.ru/signup)
   - Выполняем любые взаиможествия с сайтом: переходим по страницам, выполняем поисковые запросы и т.д., кроме описанных в двух пунктах выше
   - Хедер не должен выделятся цветом при этих взаимодействиях
